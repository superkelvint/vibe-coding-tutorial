There is no major new technical layer in this hour. The goal is to connect judgment to building. Do not pitch an app during the conversation.

<Track type="product" title="Product Judgment">

Talk to one person for 10–15 minutes about chores, responsibilities, reminders, or things people forget.

Ask:

```text
Tell me about the last time a responsibility was forgotten.

What happened?

Who was involved?

What did you do instead?

What was annoying?

How often does this happen?

What would have been good enough?
```

Fill out:

```text
User:

Situation:

Current workaround:

Pain:

Smallest useful fix:

Feature ideas to avoid for now:
```

<Response id="preamble-hour4-discovery-notes" prompt="Record the conversation using User, Situation, Current workaround, Pain, Smallest useful fix, and Feature ideas to avoid for now. Do not turn the notes into a feature pitch yet." rows={14} />

<Response id="preamble-hour4-success-signal" prompt="What observable signal would suggest the smallest useful fix helped? What evidence would still be missing after one conversation?" rows={8} />

<details>
  <summary>Hint: stay close to the person's words</summary>

  Separate what the person actually described from your interpretation. A feature idea is a hypothesis, not proof that the pain is important or frequent.
</details>

</Track>

<Track type="codex" title="Codex Prompting and Debugging">

Write three prompts. Keep the person making the product decision and require evidence in the build and debug work.

### Build prompt

Use:

```text
Goal
Context
Constraints
Acceptance criteria
Existing verification
Required new tests
Regression checks
Verification commands
Manual verification
Evidence
```

### Debug prompt

Use:

```text
Bug
Expected
Actual
Steps to reproduce
Evidence
Constraints
```

### Product discovery prompt

Use:

```text
A user asked for [feature].
Do not write code.
Help me uncover the real pain.
```

<Track type="technical" title="Technical Build">

There is no major new technical layer in this hour. Create:

```text
prework-notes.md
```

Save the discovery notes and the three prompts. Commit them as an understandable change.

Before changing the button page again, write three checks:

```text
Acceptance criterion:
Clicking the button changes the message.

Regression check:
The original message is visible before clicking.

Failure check:
If the button and JavaScript IDs do not match, the expected behavior fails and evidence appears in the Console or visible behavior.
```
