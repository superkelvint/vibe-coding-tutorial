This is a learning task, not a production setup exercise. Move slowly and ask for an explanation whenever a word or command is unfamiliar.

Git is a tool that keeps a history of the changes you make to a project.
We will use it to create clear local save points after work that we have checked.
You do not need an online account for this lesson; everything happens on your own computer.

Keep this nearby while you practice: <a href={`${import.meta.env.BASE_URL}lessons/preamble/05-preamble-git-cheatsheet/`}>Git cheat sheet: 10 commands to know</a>.

On the VS Code download page, choose the installer for your operating system. On Linux, choose the package format your distribution uses, such as `.deb` or `.rpm`. If Git is already installed, you do not need to install it again.

After installing, open a terminal and check Git with:

```bash
git --version
```

You should see a version number. You do not need to understand the version number yet; it simply confirms that the command is available.

Create a folder called:

```text
vibe-prework
```

Inside it, create:

```text
notes.txt
```

Write this exact line:

```text
My first Git save point.
```

Run each command one at a time:

```bash
git init
git status
git add notes.txt
git commit -m "Add first notes file"
git status
```

### Pause and predict

Before each command, ask:

```text
What do you think this command will change?

Does it change the file, Git's memory, or neither?
```

<Response id="preamble-hour1-command-predictions" prompt="For git init, git status, git add, commit, and the final git status, predict what changes and what evidence you expect from each command." rows={12} />

### Verify the result

Check these separately:

1. Open `notes.txt` and confirm the sentence is present.
2. Open the folder in the browser only if you have a suitable local file view; the browser is not Git and does not create commits.
3. Read the final `git status` and confirm it reports a clean working tree.

<Checkpoint id="preamble-hour1-file-check">I opened `notes.txt` and confirmed that the intended sentence is present.</Checkpoint>

<Checkpoint id="preamble-hour1-git-check">I ran the Git commands and confirmed that the final status shows the work is saved and clean.</Checkpoint>

### Explain back

You should be able to say:

```text
The file is the work.

Git noticed the file.

I chose the file for the save point.

The commit created the save point.
```

Open `notes.txt` again after the commit. Its contents should be unchanged. If the file changed unexpectedly, inspect the diff before continuing.
