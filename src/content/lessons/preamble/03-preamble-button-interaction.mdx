<Track type="codex" title="Codex Prompting and Debugging">

Use this prompt:

```text
I am a beginner.

Create one simple HTML file with:
- a message
- a button
- simple styling

When I click the button, the message should change.

Use HTML, CSS, and JavaScript in one file.
Keep it very small.
Do not use React or frameworks.

Before the code, describe the click story in plain English.

After the code, explain:
1. what the user clicks
2. what JavaScript looks for
3. what JavaScript changes
4. why the names must match
```

Before clicking, ask:

```text
What message do you expect now?

What message do you expect after clicking?

Which code probably connects the button to the message?
```

  The HTML gives the button and message identities. JavaScript must look up the same identities before it can attach a click or change the message.
</details>

<Track type="technical" title="Technical Build">

Create:

```text
button.html
```

Before clicking, record your prediction. Open the browser Console so you can inspect evidence if something fails. Click once and compare the visible result with your acceptance criteria.

Then change manually:

```text
1. Button text
2. Message before clicking
3. Message after clicking
4. One color
```

Reload after each small change and test again.

<Checkpoint id="preamble-hour3-click-tested">I tested the button before and after my manual changes and observed the intended visible message change.</Checkpoint>

### Break it on purpose

Change the button ID so the HTML and JavaScript no longer agree. Open the browser Console before asking Codex.

Ask:

```text
What happened?
What did you expect?
What evidence do we have?
Which two names should match?
```

Use this evidence-first debug prompt:

```text
Bug:
The button stopped changing the message.

Expected:
Clicking the button changes the message.

Actual:
Nothing happens.

Evidence:
[paste the browser Console error if there is one]

I intentionally changed one ID.

Help me compare:
1. the ID in the HTML
2. the ID JavaScript is looking for
3. the smallest fix

Do not rewrite the whole file.
```

<Response id="preamble-hour3-bug-report" prompt="Record the bug using Bug, Expected, Actual, Evidence, and the two names you compared. Explain why the evidence is more useful than a guess." rows={12} />

<details>
  <summary>Hint: reproduce before fixing</summary>

  Confirm that the broken behavior happens more than once, inspect the Console and source, and change only the mismatched name. A successful fix should restore the click without rewriting unrelated code.
</details>

<Response id="preamble-hour3-fix-evidence" prompt="After the smallest fix, what evidence shows the click works again? What regression did you re-test, and what remains unverified?" rows={9} />
